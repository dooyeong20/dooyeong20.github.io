1. 복잡도

1.1 측정 & 분석

  • 측정 : 알고리즘을 구동하면서 Elementary Operation의 횟수를 카운팅. 하지만 알기 쉽지 않음

  • 측정은 결과의 의미도 기준이 애매하고, 코드도 복잡해져서 잘 사용하지 않음

  • 분석 : 반복문과 조건문을 고려하여 분기별로 실행 횟수를 분석

    • 알고리즘 동작에 영향을 주는 인자를 변수로(ex: n, m, k) 사용한다.

1.2 복잡도의 종류

  • 최악의 경우
  • 최선의 경우
  • 평균적인 경우

위 세가지 모두 각각 big-O, Big-Omega, Big-Theta 다 사용할 수 있지만, 보통 최악의 경우의 Big-O 를 사용한다.

1.3 Asymptotic Notations(점진적 표현 방법)

  • 자료의 개수 등 n의 크기가 충분히 많다고 가정
  • Big-O
    • $3n+1$ 은 $O(n^2), O(n), O(n^3)$ 등 모두에게 다 포함된다. 즉, g(n)이 상한이다.
    • Big-O에 O(n)에는 {n, 4, $logn$} 등이 있을 수 있다.
    • Big-O 순위 → 상수, 로그, 선형, 선형로그($nlogn$), 다항식(제곱), 지수, 팩토리얼
    • 많이 사용하는 이유는 “최소한의 성능을 보장”한다는 느낌이다.
  • Big-Omega
    • Big-O와는 반대로, g(n)이 하한이 된다.
    • 수식을 봐도, f(n)과 c * g(n)의 위치가 바뀌어 있다.
  • Big-Theta
    • 상한, 하한 양쪽 모두에게 딱 맞아야 한다. (교집합)
    • 쉽게 말해 n의 차수가 똑같이 맞는 것만 가능하다.

2. 자료구조

2.1 추상 자료형(Abstract Data Type; ADT)

  • 정의: ‘값’과 ‘연산’의 집합으로 정의되는 논리적 행동을 가지는 오브젝트 클래스

  • 추상 자료형이란, 자료형의 수학적 모델로서, 사용자(일반 개발자) 입장에서 본 자료의 행동에 의해서 정의된다. 이 ‘행동’은 가능한 값과 가능한 연산, 그리고 이 연산의 동작으로 이루어진다.

  • 추상 자료형이 자료 구조와 다른 점은, 개발자(DataStructure 개발자) 입장의 구현 측면은 고려하지 않는다는 점이다.

  • 내부적인 구현이 빠져있어야 추상 자료형이다.

  • 예를들어, stack, queue 등의 추상 자료형의 구현체를 똑같은 이름을 그대로 사용해서 자료구조로 구현할 수도 있다.

  • 사용과 개발의 분리를 위해서 추상 자료형이 쓰인다.

2.2 배열 (Arrays)

  • 배열은 자료구조이며, 배열에서 인덱스는 유일무이한 식별자(Identifier)로 사용
  • 하나의 변수에 여러 자료를 저장
  • 배열을 만들 때는 Type과 Length가 필요하다.

2.2.1 배열의 특징

  • 크기(Element의 개수)가 정해져 있다. ⭐
  • 자료 구조에 기능(메소드)이 포함되어 있지 않다.
  • 자료가 메모리상에 빈틈 없이 연속적으로 위치해 있다. ⭐
  • 인덱스를 활용하여 자료에 빠르게 접근(임의 접근, random access)할 수 있다. ⭐
    • RAM(Random Access Memory) : 임의적으로 접근이 가능함
  • 배열의 길이는 생성 시 정해져, 변경할 수 없다.
  • 가변 길이 배열은 배열의 크기를 변경할 때 마다 새 배열을 만든다.
  • 실제로는 Element의 삭제가 불가능

3. 리스트(Lists)

기본적으로 추상 자료형(ADT)⭐이며, 순차적인 데이터(Sequential Data)를 다룸

인덱스가 있지만, 배열(Array)의 index와 의미가 약간 다름

3.1 연산자

  • 리스트를 생성
  • 리스트가 비어있는지 확인
  • 리스트의 앞에 개체를 삽입(prepending)
  • 리스트 뒤에 개체를 삽입(appending)
  • 리스트의 첫 머리(head)를 결정
  • 주어진 인덱스에 해당하는 요소에 접근
    • 내부를 array로 구현했으면 random access가 가능함
  • 주어진 인덱스에 새로운 요소를 삽입
  • 주어진 인덱스에 해당하는 요소를 제거

3.2 배열 리스트(Array List)

내부 저장소로써 배열을 사용해 구현하는 List

  • is_empty(): O(1) - length를 사용할 수 있기 때문

  • prepend(): O(n)

  • append(): O(n) (조건부 O(1)) - 배열의 크기를 넘었을 때 다시 큰 크기로 재할당 해야하기 때문에

  • set_head(index)

    : O(1) ⭐ - 시작점을 바꿔주고 길이만 바꿔주면 되기 때문

    • set_head 에서 self.array[index:] 는 실제 python 슬라이싱이 아니라 C 수준으로 최적화가 되어 있어서

  • access(index): O(1)

  • insert(item, index): O(n) - 최악(가장 앞에 넣는 경우)

  • remove(index): O(n) - 최악(가장 앞의 요소를 제거하는 경우)

3.2 연결 리스트(Linked List)

단방향, 양방향 리스트가 존재함

  • is_empty(): O(1)
  • prepend(): O(1)
  • append(): O(1) (w/ access: O(n)) ⭐ - 노드를 찾아가는데에 O(n)
  • set_head(index): O(1) (w/ access: O(n)) ⭐- 노드를 찾아가는데에 O(n)
  • access(index) : O(n)
  • insert(item, index): O(1) (w/ access: O(n)) - 노드를 찾아가는데에 O(n)
  • remove(index): O(1) (w/ access: O(n)) - 노드를 찾아가는데에 O(n)

[자료구조]

  • 자료(Data): 현실 세계로부터 수집한 사실이나 개념의 값 또는 이들의 집합. 특정 용도로 사용하기 위해 처리/가공한 것을 정보(Information)이라 한다.
  • 자료구조(Data Structure): 자료구조는 자료 값의 모임, 자료간의 관계, 그리고 자료에 적용할 수 있는 함수나 명령을 의미

1. 자료구조의 특징

  • 효율성
  • 추상화
  • 재사용성

2. 자료구조 종류

  • 선형(Linear - 일자로 되어있다, 여러갈래로 갈라지지 않는다)
    • 스택과 큐는 배열/연결 리스트를 효율성을 극대화 해놓은 자료구조이다.
    • 배열/연결 리스트, 스택, 큐
  • 비선형 (방향성 존재, 여러 갈래로 갈라질 수 있다)
    • 트리, 그래프
    • 어떠한 자료구조도 다 포함할 수 있음

3. 자료구조의 필요성

  • 효율적인 접근 및 사용(중복 제거)
  • 적절한 알고리즘을 기계적으로 적용
  • 협업에 큰 도움(설명같은것도 덜 필요함)

4. Python의 자료구조

  • List(Queue, Stack)
  • Map(dictionary)
  • Set
  • Heap

[알고리즘]

: 알고리즘이란? 문제 해결을 위한 여러 동작들의 모임

1. 알고리즘 조건

  • 입력 : 외부에서 제공되는 자료가 존재해야함
  • 출력 : 2가지 이상의 다른 결과를 출력
  • 명확성 : 명확한 명령어로 구성해야함 (코드로 작성할 수 있을 만큼 명확)
  • 유한성 : 유한한 시간안에 종료되어야함. 그래야 출력이 나오니까.
  • 효과성 : 사람이 종이와 연필로 수행할 수 있을 정도로 단순

2. 알고리즘 필요성

  • 서비스의 규모
  • 컴퓨터 연산 & 비용 (클라우드)

3. Python

  • glue 언어 : 다른 언어로 구현된 걸 풀로 붙이듯 사용할 수 있음
  • OOP, FP(람다 문법을 사용) 모두 지원
  • 코딩테스트에서 Comprehension, lambda 등 간결한 코드 작성 가능

[코딩테스트]

1. 목적

  • 빠르고 효율적으로 실력 가늠
  • 코딩 습관
  • 공통 역량 준비 가능

2. 유형

  • 기본 라이브러리 효율적 구현 - 예외처리로 test case들 해결
  • 자료구조 - 자료구조를 잘 선택 (dictoinary, set, heap 등)
  • 알고리즘 - sorting, DP, DFS/BFS, greedy, Dijkstra
    • 알고리즘을 모르면 절대로 못 푸는 문제

[알고리즘 복잡도]

  1. 공간 복잡도 vs. 시간 복잡도 일반적으로 두 복잡도는 Trade-off 관계
    • 공간복잡도 : 알고리즘이 동작하기 위해 필요한 공간(메모리)의 크기
    • 시간복잡도 : 알고리즘이 동작하는 데에 걸리는 시간또는 연산의 횟수(기기마다 스펙이 다르기 때문에 시간은 잘 안쓰고 보통 횟수를 많이 사용함)

Problem Solving - 카드 정렬하기(G4)

1. 문제 [ Link ]

정렬된 두 묶음의 숫자 카드가 있다고 하자. 각 묶음의 카드의 수를 A, B라 하면 보통 두 묶음을 합쳐서 하나로 만드는 데에는 A+B 번의 비교를 해야 한다. 이를테면, 20장의 숫자 카드 묶음과 30장의 숫자 카드 묶음을 합치려면 50번의 비교가 필요하다.

매우 많은 숫자 카드 묶음이 책상 위에 놓여 있다. 이들을 두 묶음씩 골라 서로 합쳐나간다면, 고르는 순서에 따라서 비교 횟수가 매우 달라진다. 예를 들어 10장, 20장, 40장의 묶음이 있다면 10장과 20장을 합친 뒤, 합친 30장 묶음과 40장을 합친다면 (10 + 20) + (30 + 40) = 100번의 비교가 필요하다. 그러나 10장과 40장을 합친 뒤, 합친 50장 묶음과 20장을 합친다면 (10 + 40) + (50 + 20) = 120 번의 비교가 필요하므로 덜 효율적인 방법이다.

N개의 숫자 카드 묶음의 각각의 크기가 주어질 때, 최소한 몇 번의 비교가 필요한지를 구하는 프로그램을 작성하시오.

2. 입력

첫째 줄에 N이 주어진다. (1 ≤ N ≤ 100,000) 이어서 N개의 줄에 걸쳐 숫자 카드 묶음의 각각의 크기가 주어진다. 숫자 카드 묶음의 크기는 1,000보다 작거나 같은 양의 정수이다.

4. 풀이
