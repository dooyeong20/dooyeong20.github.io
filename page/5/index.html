HTML 표 컨텐츠와 양식

표 컨텐츠

table, tr, th, td

  • table: display=table
  • tr: display=table-row
  • th,td: display=table-cell
  • th는 테이블의 헤더 역할을 한다(글씨가 두껍게 되고 등등)

th

  • abbr : 열에 대한 간단한 소개
  • headers: 관련된 하나 이상의 다른 머리글 칸 id속성 값
  • colspan
  • rowspan
  • span: 자신이 누구 머리글 칸인지 명시 (col, colgroup, row, rowgroup, auto(row))

td

  • headers : 보통 th의 id에 서연결한다.
  • colspan
  • rowspan

caption

  • display는 table-caption이다
  • 테이블 태그 바로 하위에 작성하며 테이블 제목의 역할을 수행한다

양식 태그

<form>

웹 서버에 정보를 제출하기위한 양식의 범위를 정함

  • action: 전송한 정보를 처리할 url
  • autocomplete: 사용자가 이전에 입력한 값으로 자동완성 할지 여부
  • method: 서버로 전송할 방식
  • novalidate: 유효성을 검사하지 않도록 지정
  • target: 서버로 전송 후 응답받을 방식을 지정

<input>

사용자에게 입력받을 데이터 양식

  • autofocus: 페이지가 로드될 때 자동으로 포커스

  • form: 폼 바깥에 작성한 것을 이어주는 역할

  • type: 입력할 수 있는 종류

    • button
    • checkbox
    • color 등
    • 예를 들어서,type=number과 같이 숫자를 입력하려고 했으면 숫자만 입력이 됨
    • image 이미지로 끝나지 않고, 제출 버튼으로 활용할 수 있다.
    • reset 데이터는 자신이 포함된 form태그 내의 input들을 초기화한다.

  • placeholder: 사용자에게 힌트를 줌

<label>

라벨 가능 요소(labelable)의 제목(Caption).

  • for 속성으로 라벨 가능 요소를 참조하거나 콘텐츠로 포함.
  • 라벨 가능 요소: <button>, <input>, <progress>, <select>, <textarea>

<button>

  • autofocus 페이지가 로드될 때 자동으로 포커스
  • disabled 버튼을 비활성화
  • form <form>의 id 속성 값
  • name 폼 데이터와 함께 전송되는 버튼의 이름
  • type 버튼의 타입
    • button, submit, reset

<textarea>

여러 줄의 일반 텍스트 양식

  • display: inline-block
  • rows 양식의 줄 수 숫자(Number)
    • default: 2

<fieldset>, <legend>

같은 목적의 양식을 그룹화(<fieldset>)하여 제목(<legend>)을 지정

멀티미디어 & 내장 콘텐츠 Tag

<img> Tag

이미지 삽입을 위한 태그

  • src: (필수)이미지 URL
  • alt: (필수)이미지의 대체텍스트
  • width: 이미지의 가로 너비 (px 없이 숫자만)
  • height: 이미지의 세로 너비 (px 없이 숫자만)
  • srcset: 브라우저에게 제시할 이미지 URL과 원본 크기의 목록을 정의
  • sizes: 미디어 조건과 해당 조건일 때 이미지 최적화 크기의 목록을 정의

srcset

srcset & sizes link

브라우저에 제시할(사용할) 이미지들과 그 이미지들의 원본 크기를 지정

  • 사이즈별로 2장 이상 준비하여 srcset 속성에 작성
  • px단위가 아닌 w 디스크립터 혹은 x 디스크립터를 입력해야 하며, 작은 크기 이미지부터 순서대로 입력
  • 늘려서 사용하는 것 보다는 줄여서 사용할 수 있는 가장 가까운 크기의 사진을 선택

W unit

Width descriptor)는 이미지의 원본 크기(가로 너비)를 의미

  • 400x300(px) 크기 이미지의 w 값은 400w
  • 고정된 이미지 크기를 유지하려면 width 속성을 추가
    sizes 속성과는 다른 개념

X unit

x 디스크립터(Device pixel ratio descriptor)는 이미지의 비율 의도

  • 디바이스의 픽셀 비율(Device pixel ratio)과 일치하는 값으로 최적화 선택
  • 정수(integer) 값으로 제공하는 것이 좋음

2

sizes

미디어조건과 그 조건에 해당하는 이미지의 ‘최적화 출력 크기’ 를 지정

결론

srcset & sizes link

  • width는 이미지의 ‘출력 크기’만 지정
  • sizes는 이미지의 ‘출력 크기’ + ‘최적 크기’도 함께 지정
  • 따라서 sizes=”500px”이 지정된 첫번째 예제는 500px에 최적화된 이미지로 heropy_medium.png를 사용했고 이미지 크기도 500px로 설정
  • sizes와 width를 같이 작성할 경우 width가 우선
  • srcset과 sizes는 IE를 지원하지 않음

A Tag

다른 페이지, 같은 페이지 위치(#, 해시 태그), 파일, 이메일 주소, 전화번호 등 다른 URL로 넘어가는 하이퍼링크, Anchor

속성들

  • rel: 현재 문서와 URL강늬 관계 (license, prev, next 등)
  • target: 링크 URL 위치 (_self, _blank)
  • type: 해당 URL의 MIME 타입 (test/html)

HTML 콘텐츠 구분

BODY 내 구조 개념

body 태그는 정보의 구조들을 나타냄

  • 의미가 없는 div 태그들로 도배하면 좋지 않다.
  • header 태그, ul, li 등 의미를 가지는 태그들을 사용해야 한다.
  • 일반적으로 영역을 나타내는 태그들은 block요소이다

콘텐츠 구분

Header Tag

소개나 탐색을 돕는 그룹을 나타냄

  • 로고
  • 검색바
  • 메뉴 등

header나 footer는 후손이 될 수 없다.

Footer Tag

작성자 구획, 저작권 데이터, 관련 문서 등에 대한 링크, 정보를 포함한다.
일반적으로 사이트 하단의 정보들을 포함

header나 footer는 후손이 될 수 없다.

H Tag

6단계의 문서 제목을 지칭
h1 ~ h6까지 존재

  • 크기 조절을 위해서 h 태그를 사용하는 것은 좋지 않고, 대신 css의 font-size 속성을 사용해야 함
  • 제목 단계를 건너뛰면 좋지 않음
  • h1은 한 페이지에 하나씩만 사용해야 함
  • 추천하는 방식정도

Main Tag

<body> 의 주요 콘텐츠를 나타내는 태그
문서 핵심 주제 및 핵심 기능성 등과 직접적으로 연관된 콘텐츠들로 구성

  • 문서에 하나의 <main>만 존재해야 함
  • IE에서 지원하지 않음

Article Tag

독립적으로 구분되거나 재사용 가능한 영역을 설정
매거진/신문 기사, 블로그 등

  • 일반적으로 <h1> ~ <h6>를 포함하여 식별.
  • 작성일자와 시간을 <time>의 datetime 속성으로 작성.
  • article 태그 안에 section 태그가 있어도 되고, 그 반대도 가능함

Section Tag

문서의 일반적인 영역을 설정

  • 영역을 의미하기 때문에 일반적으로 <h1> ~ <h6>를 포함하여 식별
  • div와 차이: 의미가 있기 때문에 h태그로 제목을 포함함
  • 독림성을 가지는 article 태그와 함께 사용할 수 있음

Aside Tag

문서의 별도 콘텐츠를 설정

  • 보통 광고나 기타 링크 등의 사이드바를 설정

Nav Tag

다른 페이지 링크를 제공하는 영역

  • Navigation, 보통 메뉴(Home, About, Contact), 목차, 색인
  • ol, ul 태그와 함께 메뉴들을 보여줄 때 자주 사용됨

Address Tag

<body>, <article>, <footer> 등에서 연락처 정보를 제공하기 위해 포함

  • block 요소

Div Tag

본질적으로 아무것도 나타내지 않는 콘텐츠 영역

  • Division, 꾸미는 용도

문자 구분

ol, ul, li Tag

각 항목<li> 의 정렬된 목록<ol>이나 정렬되지 않은 목록<ul>을 설정

  • <ol>과 <ul>은 자식으로 <li>만 포함 가능
  • <li>는 단독으로 사용할 수 없으며, <ol>이나 <ul>에 자식으로 포함되어야 함
  • 정렬된 목록(<ol>)의 항목 순서는 중요도를 의미할 수 있음

ol 속성

  • start: 항목에 매겨지는 번호의 시작 값 숫자(Number)
  • type: 항목에 매겨지는 번호의 유형 a, A, i, I, 1
  • reversed: 항목의 순서만 바뀌고 물리적 위치는 바뀌지 않음

li 속성

  • value: 항목의 순서를 설정 숫자(Number) 이하 항목들의 순서가 다시 지정됨

dl, dt, dd

용어(<dt>)와 정의(<dd>) 쌍들의 영역(<dl>)을 설정

  • block 요소
  • Description List, Definition Details, Definition Term
  • <dl>은 <dd>, <dt>만을 포함해야 함
  • 키(key)/값(value) 형태를 표시할 때 유용
  • 근데 dl 태그안에 div 와 같은 태그를 넣을 수 없기 때문에 스타일링하기 힘듬

HTML 기본 문법

작성일 2021-04-12 Edited on 2021-05-05 In HTML/CSS , Study

HTML 문법

기본 형태

요소, 태그로 이루어져 있음. 태그와 요소는 거의 같은 것

  • 태그는 열리고(open) 닫히는(close) 태그 구조를 가지고 있으며 이는 한 쌍

속성과 값

태그(요소)의 기능을 확장하기 위해 ‘속성’을 사용

  • <img />는 이미지를 삽입할 때 사용하는 태그
  • src(source)라는 속성을 사용해서 삽입할 이미지의 경로를 지정
  • alt(alternative)라는 속성은 이미지를 출력하지 못하는 상황에 이미지 대신 보여질 텍스트를 지정
  • <div></div>는 의미를 가지지 않는 태그
  • 닫히는 태그가 없으면 빈 태그(Empty Tag)

부모, 자식 요소

태그A가 태그B의 콘텐츠로 사용되면, 태그B는 태그A의 부모 요소, 태그A는 태그B의 자식 요소

  • <ul>은 <section>의 자식 요소이면서 <li>의 부모 요소
  • 부모보다 높다면 조상(상위)요소

HTML, CSS, JavaScript

웹에서 동작하는 3가지 언어들

HTML

페이지에 제목, 문단, 표 이미지 등을 정의하고 구조와 의미를 부여하는 정적인 언어

CSS

Cascading Style Sheets 콘텐츠 구조를 꾸며주는 언어 웹의 시작적 표현을 담당.

JavaScript

컨텐츠를 동적으로 꾸며주는 역할을 하는 언어

크루스칼 알고리즘

작성일 2021-04-09 Edited on 2021-05-01 In DS & Algorithm

Kruskal’s Algorithm

어느 그래프의 Subgraph 중, 다음 조건을 만족하는 그래프

  1. 본 그래프의 모든 노드를 포함
  2. 모든 노드가 서로 연결 (Connected Graph)
  3. 사이클이 존재하지 않음 (Acyclic Graph)

위 조건들을 만족하는 그래프 중 간선 가중치의 합이 최소인 그래프

  • 무향 그래프일 때 동작함
  • 미로 생성에도 사용됨

알고리즘 동작

그리디하게 동작하는 알고리즘

  1. 모든 간선을 가중치를 기준으로 오름차순 정렬 (O(eloge))
  2. 작은 가중치의 간선부터 하나씩 선택하여 MST를 구성한다.
  3. 이 과정에서 사이클을 형성하는 간선은 선택하지 않는다.
  4. 사이클 형성을 판단하는 알고리즘은 Union-Find 알고리즘

union find

Union

각 노드들이 집합에 포합되어 있다고 가정하고, 같은 집합에 포함시킴

  • 각 노드들이 들어있는 집합의 루트노드를 다른 집합의 루트를 바라보게

Union 최적화

  • union-by-rank
    • rank가 증가하는 속도를 O(logN)으로 줄여주는 효과

Find

각 노드들의 집합의 루트 노드를 찾는 과정이다

Find 최적화

  • Path Comprehension
    • memoization으로 됨

최적화

유니온과 파인드 둘다 최적화를 함으로써 최적의 결과를 얻을 수 있다.
해당 알고리즘의 시간 복잡도는 따라서, O(eloge) + O(v)가 된다.

Prim Algorithm

크루스칼과는 다르게, vertex를 기준으로 인접 노드들을 하나씩 추가해나가는 방법

  • 하나에서부터 점점 확장해나가는 방식이라고 할 수 있음
    • 그래서 union find가 필요없이 현재 집합에 있는지 여부로만 구현 가능
  • 선택한 노드로부터 가장 낮은 가중치를 갖는 간선으로 이어지는 노드 선택
    • 이 과정을 모든 노드가 선택될때까지 반복 (O(V^2))
    • 최적화 알고리즘 적용 시 시간복잡도는 O(ElogV)

Kruscal vs Prim

  • O(ElogE) vs O(ElogV)
  • sparse 하면 크루스칼 사용
  • dense 하면 프림 사용
