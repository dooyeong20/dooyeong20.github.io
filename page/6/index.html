Problem Solving - 영역 구하기(S1)

1. 문제 [Link]

눈금의 간격이 1인 M×N(M,N≤100)크기의 모눈종이가 있다. 이 모눈종이 위에 눈금에 맞추어 K개의 직사각형을 그릴 때, 이들 K개의 직사각형의 내부를 제외한 나머지 부분이 몇 개의 분리된 영역으로 나누어진다.
M, N과 K 그리고 K개의 직사각형의 좌표가 주어질 때, K개의 직사각형 내부를 제외한 나머지 부분이 몇 개의 분리된 영역으로 나누어지는지, 그리고 분리된 각 영역의 넓이가 얼마인지를 구하여 이를 출력하는 프로그램을 작성하시오.

2. 입력

첫째 줄에 M과 N, 그리고 K가 빈칸을 사이에 두고 차례로 주어진다. M, N, K는 모두 100 이하의 자연수이다. 둘째 줄부터 K개의 줄에는 한 줄에 하나씩 직사각형의 왼쪽 아래 꼭짓점의 x, y좌표값과 오른쪽 위 꼭짓점의 x, y좌표값이 빈칸을 사이에 두고 차례로 주어진다. 모눈종이의 왼쪽 아래 꼭짓점의 좌표는 (0,0)이고, 오른쪽 위 꼭짓점의 좌표는(N,M)이다. 입력되는 K개의 직사각형들이 모눈종이 전체를 채우는 경우는 없다.

3. 예제 입/출력

4. 풀이

주어진 사각형들의 꼭지점 정보들을 바탕으로 먼저 사각형을 2차원 배열에 칠한 후, 해당 부분을 제외한 나머지 빈 공간을 bfs로 탐색해주며 각각의 넓이와 빈 공간의 개수를 탐색해주었다.

이 때 조심해야 하는 것은, 주어진 꼭짓점의 좌표 특징은 x축 좌-> 우, 그리고 y축 하 -> 상의 방향으로 주어져있기 때문에 배열에 인덱스로 대입할 때 주의해야 한다.

이진 탐색 트리(Binary Search Trees)

  • 노드 값에 ‘순서’, ‘크고 작음’이 있어서 비교를 할 수가 있다.(알파벳, 숫자 등도 포함)
  • 왼쪽 서브트리가 더 작고 오른쪽 서브트리 요소(노드)들이 더 크다.
  • 자료를 넣는 순서에 영향을 받는다.
  • O(logN) ~ O(n) 사이 어딘가에 존재하게 된다.
  • 자료의 중복이 허용되지 않음
  • (굳이 따지자면) 탐색 최적화

자료의 탐색

  • Skewed 경우 : O(n)
  • Balance 경우 : O(logN)

자료의 삽입

  • 일치하는 노드가 발견되면 종료
  • 일치하는 노드가 없으면 마지막 노드의 좌(작을 때), 우(클 때)에 삽입.

자료의 삭제

세가지 경우가 존재함
삭제를 하기 위해 search를 할 때는 부모노드를 추적해야 한다.

  1. 삭제할 노드의 자식이 없을 때
  2. 삭제할 노드가 좌/우 중 하나의 자식이 있는 경우
  3. 삭제한 노드가 좌, 우 자식 모두 가지고 있는 경우 (매우 복잡)

그래프

객체간의 짝을 이루는 구조를 나타냄. 가장 유연하다 (ADT 임)

  • vertex와 이를 잇는 edge로 이루어져 있다.
  • 방향이 있거나(유유) 없을(무향) 수 있다.
  • edge에는 가중치라는 것이 있다.(클수록 연결의 강도가 클 수도있고, 작아질 수도 있다.)

그래프와 트리 관계

트리는 DAG(Directed Acyclic Graph)의 일종이며 회로(사이클)이 없고, 방향성이 존재한다.

그래프의 종류

  • 무향 그래프(양쪽으로 다 연결되어 있는 것으로 생각할 수 있다.)
  • 유향 그래프
    • 같은 노드를 잇더라도 방향에 따라서 가중치가 다를 수 있다.
  • 다중 그래프
    • 하나의 vertex에서 다른 vertex로 가는 edge가 여러 개 있을 수 있다
    • 자기 자신으로 연결되는 edge가 있을 수 있다.

그래프 용어

  • path: edge들의 시퀀스
  • 회로(Cycle, Circuit): 트리와는 다르게 사이클이 존재할 수 있다. 자기 자신으로 다시 돌아올 수 있음.
  • 부분 그래프(Subgraph): 그래프에 속하는 그래프
    • 모든 노드끼리 연결이 안되어 있어도 그래프임 즉, 1-7-3 6 4 이렇게 (1,7,3),(6), (4) 이렇게 구성해도 connected graph가 아닌 subgraph이다.
  • 연결 그래프(connected graph): 임의의 노드와 임의의 노드를 연결하는 edge가 있다. 즉, 어떻게든 path를 구성해서 노드간에 이동할 수 있다.

그래프 구현 방법

  • 인접 리스트: Sparse 할 경우 유리함
    • 각 vertex마다 연결된 vertex들을 쭉 리스트로 나열함
    • 인접 리스트에서 순서는 중요하지 않음
      • 트리는 left, right(순서)가 중요하지만 그래프는 중요하지 않음
  • 인접 행렬 - Dense할 경우 유리함
    • 노드들간에 연결이 다 되어 있을 수록 유리
    • 무향 그래프인 경우, 인접 행렬에서 row=colum 축에 대칭임
    • 가중치가 있을 경우 (v1, v2)에 1대신에 가중치를 적음
    • 행렬이 값으로 차있을 경우(dense) 효율적

트리

작성일 2021-03-26 Edited on 2021-05-01 In DS & Algorithm

이전 복습

  • Composition -> ex) ComposedLinkedQueue라는 클래스가 DoublyLinkedList를 사용함
  • 생명주기를 함께 함

1. 트리 - 비선형 자료구조

  • 그래프의 일종, 여러 노드가 한 노드를 가르킬 수 없음(방향성이 있다고 보는 것(부모가 자식을 가르키는 구조라는 개념))
    • 즉, 부모는 없거나(루트 노드) 하나밖에 없다
    • root는 부모가 없기 때문에 자기 자신을 가르켜도 된다(?)
  • 회로가(사이클이) 없고, 두 노드를 잇는 길이 하나뿐임
  • root node가 하나 있고, 자식 노드가 있는 것이다
    • 이 때, 자식의 수는 0 ~ n개가 있을 수도 있다

1.1 트리의 구조

선형이 아니기 때문에 다양한 용어들이 존재함

  • Root node : 부모가 없는, 뿌리 노드
  • leaf node : 자식이 없는 노드(leaf node간에도 level의 차이가 있을 수 있음)
  • level(depth): root가 level 0, root부터의 거리를 level 을 정함
  • internal node :star: : leaf node를 제외한 나머지 노드들(root node 포함) :star:
  • height : tree에서 가장 높은 레벨
  • edge : 노드들을 연결하는 간선
  • path : :star: 경로를 따질때는 노드들은 빠지고 edge들만 따진다 :star:

1.2 트리의 특징

  • 하나의 노드에서 다른 노드로 이동하는 경로는 유일하다 :star:
  • Acyclic(사이클이 없다)
  • 모든 노드는 서로 연결됨(외딴 섬이 없다)
    • 따라서 하나의 edge를 끊으면 트리가 두개가 됨 (두 개의 sub-tree로 분리)
    • leaf node만 잘라내도 leaf node가 root node가 되는 tree가 된다
  • edge = node - 1

2. 이진 트리(Binary Tree) 종류

자식이 최대 2개 있을 수 있는 트리
왼쪽, 오른쪽을 나누기 때문에 경우의 수는 총 4개이다.
자식 있을 경우, 없을 경우, 왼쪽만, 오른쪽만

2.1 정 이진 트리(full binary tree, Perfect binary tree)

꽉 차있는 binary tree

  • 모든 리프노드들의 level이 같음
  • leaf node들을 제외한 모든 노드들이 자식 노드가 2개씩 있음

2.2 완전 이진 트리(Complete binary tree)

가장 흔하고 많이 다루는 트리
Perfect vs Complete (정 이진트리 vs 완전 이진 트리)

  • height에 레벨 끝에서 2번째까지는 완전 다 차있음
  • leaf node들은 :star: 왼쪽에서부터 오른쪽으로 :star: 차례대로 차있는 트리

2.3 군형 이진 트리(Balnaces binary tree vs Skewed Tree)

  • Skewed : 한쪽으로 치우친 트리
  • balance : 밸런스도 기준이 다양함
    • 좌우 height 차이가 1 이하
    • 좌우 node 개수 차이가 3 이하

3. 이진 트리의 순회(Traversal) (보통 재귀로 구현)

목적은 잘 모르겠고 말그대로 순회하는 것임

  • (전위) 깊이 우선 순회(:star:Preorder, DFS:star:)
    • node(root) -> left -> right
  • (중위) 대칭 순회 (:star: Inorder, Symmetric)
    • left -> node(root) -> right
  • 후위 순회 (:star: Postorder :star:)
    • left -> right -> node(root)

4. 이진 트리의 탐색(Search)

단순 순회가 아니라 특정한 값을 :star: 찾기 위한 것 :star:.

  • 너비 우선 (BFS)

    • 같은 level인 애들을 차례대로 봄
    • 루트 노드에서 가장 가까운 노드를 찾는다

  • 깊이 우선 (DFS)

    • depth가 증가하는 방향으로 이동하며 Search 함

1. 스택(Stack)

스택은 후입선출(LIFO)의 특성을 가지는 추상 자료형(ADT)이다.

2. 큐(Queue)

큐는 FIFO 특성을 가지는 추상 자료형(ADT)이다.

만약 서버가 요청을 처리하는 속도와 요청하는 속도가 똑같으면 큐가 터져버린다.

2.1 큐의 연산

  • put(enqueue)
  • get(dequeue)
  • peek

2.2 큐의 구현

  • 선형 큐(Linear Queue)
    • 선형 큐는 공간의 낭비가 된다(언젠가 rear가 배열의 끝에 도착)
  • 환형 큐(Circlular Queue)
    • 큐의 용량을 전부 사용하기 전까지는 overflow가 발생하지 않음
  • 연결리스트 큐(Linked Queue)
    • 보통 양방향 연결리스트로 큐를 구성
    • 물리적인 메모리를 전부 사용하기 전에는 overflow가 발생하지 않음

시간 복잡도와 배열

작성일 2021-03-26 Edited on 2021-05-01 In DS & Algorithm

1. 복잡도

1.1 측정 & 분석

  • 측정 : 알고리즘을 구동하면서 Elementary Operation의 횟수를 카운팅. 하지만 알기 쉽지 않음

  • 측정은 결과의 의미도 기준이 애매하고, 코드도 복잡해져서 잘 사용하지 않음

  • 분석 : 반복문과 조건문을 고려하여 분기별로 실행 횟수를 분석

    • 알고리즘 동작에 영향을 주는 인자를 변수로(ex: n, m, k) 사용한다.

1.2 복잡도의 종류

  • 최악의 경우
  • 최선의 경우
  • 평균적인 경우

위 세가지 모두 각각 big-O, Big-Omega, Big-Theta 다 사용할 수 있지만, 보통 최악의 경우의 Big-O 를 사용한다.

1.3 Asymptotic Notations(점진적 표현 방법)

  • 자료의 개수 등 n의 크기가 충분히 많다고 가정
  • Big-O
    • $3n+1$ 은 $O(n^2), O(n), O(n^3)$ 등 모두에게 다 포함된다. 즉, g(n)이 상한이다.
    • Big-O에 O(n)에는 {n, 4, $logn$} 등이 있을 수 있다.
    • Big-O 순위 → 상수, 로그, 선형, 선형로그($nlogn$), 다항식(제곱), 지수, 팩토리얼
    • 많이 사용하는 이유는 “최소한의 성능을 보장”한다는 느낌이다.
  • Big-Omega
    • Big-O와는 반대로, g(n)이 하한이 된다.
    • 수식을 봐도, f(n)과 c * g(n)의 위치가 바뀌어 있다.
  • Big-Theta
    • 상한, 하한 양쪽 모두에게 딱 맞아야 한다. (교집합)
    • 쉽게 말해 n의 차수가 똑같이 맞는 것만 가능하다.

2. 자료구조

2.1 추상 자료형(Abstract Data Type; ADT)

  • 정의: ‘값’과 ‘연산’의 집합으로 정의되는 논리적 행동을 가지는 오브젝트 클래스

  • 추상 자료형이란, 자료형의 수학적 모델로서, 사용자(일반 개발자) 입장에서 본 자료의 행동에 의해서 정의된다. 이 ‘행동’은 가능한 값과 가능한 연산, 그리고 이 연산의 동작으로 이루어진다.

  • 추상 자료형이 자료 구조와 다른 점은, 개발자(DataStructure 개발자) 입장의 구현 측면은 고려하지 않는다는 점이다.

  • 내부적인 구현이 빠져있어야 추상 자료형이다.

  • 예를들어, stack, queue 등의 추상 자료형의 구현체를 똑같은 이름을 그대로 사용해서 자료구조로 구현할 수도 있다.

  • 사용과 개발의 분리를 위해서 추상 자료형이 쓰인다.

2.2 배열 (Arrays)

  • 배열은 자료구조이며, 배열에서 인덱스는 유일무이한 식별자(Identifier)로 사용
  • 하나의 변수에 여러 자료를 저장
  • 배열을 만들 때는 Type과 Length가 필요하다.

2.2.1 배열의 특징

  • 크기(Element의 개수)가 정해져 있다. ⭐
  • 자료 구조에 기능(메소드)이 포함되어 있지 않다.
  • 자료가 메모리상에 빈틈 없이 연속적으로 위치해 있다. ⭐
  • 인덱스를 활용하여 자료에 빠르게 접근(임의 접근, random access)할 수 있다. ⭐
    • RAM(Random Access Memory) : 임의적으로 접근이 가능함
  • 배열의 길이는 생성 시 정해져, 변경할 수 없다.
  • 가변 길이 배열은 배열의 크기를 변경할 때 마다 새 배열을 만든다.
  • 실제로는 Element의 삭제가 불가능

3. 리스트(Lists)

기본적으로 추상 자료형(ADT)⭐이며, 순차적인 데이터(Sequential Data)를 다룸

인덱스가 있지만, 배열(Array)의 index와 의미가 약간 다름

3.1 연산자

  • 리스트를 생성
  • 리스트가 비어있는지 확인
  • 리스트의 앞에 개체를 삽입(prepending)
  • 리스트 뒤에 개체를 삽입(appending)
  • 리스트의 첫 머리(head)를 결정
  • 주어진 인덱스에 해당하는 요소에 접근
    • 내부를 array로 구현했으면 random access가 가능함
  • 주어진 인덱스에 새로운 요소를 삽입
  • 주어진 인덱스에 해당하는 요소를 제거

3.2 배열 리스트(Array List)

내부 저장소로써 배열을 사용해 구현하는 List

  • is_empty(): O(1) - length를 사용할 수 있기 때문

  • prepend(): O(n)

  • append(): O(n) (조건부 O(1)) - 배열의 크기를 넘었을 때 다시 큰 크기로 재할당 해야하기 때문에

: O(1) ⭐ - 시작점을 바꿔주고 길이만 바꿔주면 되기 때문

  • set_head 에서 self.array[index:] 는 실제 python 슬라이싱이 아니라 C 수준으로 최적화가 되어 있어서

  • access(index): O(1)

  • insert(item, index): O(n) - 최악(가장 앞에 넣는 경우)

  • remove(index): O(n) - 최악(가장 앞의 요소를 제거하는 경우)

3.2 연결 리스트(Linked List)

단방향, 양방향 리스트가 존재함

  • is_empty(): O(1)
  • prepend(): O(1)
  • append(): O(1) (w/ access: O(n)) ⭐ - 노드를 찾아가는데에 O(n)
  • set_head(index): O(1) (w/ access: O(n)) ⭐- 노드를 찾아가는데에 O(n)
  • access(index) : O(n)
  • insert(item, index): O(1) (w/ access: O(n)) - 노드를 찾아가는데에 O(n)
  • remove(index): O(1) (w/ access: O(n)) - 노드를 찾아가는데에 O(n)

자료구조와 알고리즘이란

작성일 2021-03-26 Edited on 2021-05-01 In DS & Algorithm

[자료구조]

  • 자료(Data): 현실 세계로부터 수집한 사실이나 개념의 값 또는 이들의 집합. 특정 용도로 사용하기 위해 처리/가공한 것을 정보(Information)이라 한다.
  • 자료구조(Data Structure): 자료구조는 자료 값의 모임, 자료간의 관계, 그리고 자료에 적용할 수 있는 함수나 명령을 의미

1. 자료구조의 특징

  • 효율성
  • 추상화
  • 재사용성

2. 자료구조 종류

  • 선형(Linear - 일자로 되어있다, 여러갈래로 갈라지지 않는다)
    • 스택과 큐는 배열/연결 리스트를 효율성을 극대화 해놓은 자료구조이다.
    • 배열/연결 리스트, 스택, 큐
  • 비선형 (방향성 존재, 여러 갈래로 갈라질 수 있다)
    • 트리, 그래프
    • 어떠한 자료구조도 다 포함할 수 있음

3. 자료구조의 필요성

  • 효율적인 접근 및 사용(중복 제거)
  • 적절한 알고리즘을 기계적으로 적용
  • 협업에 큰 도움(설명같은것도 덜 필요함)

4. Python의 자료구조

  • List(Queue, Stack)
  • Map(dictionary)
  • Set
  • Heap

[알고리즘]

: 알고리즘이란? 문제 해결을 위한 여러 동작들의 모임

1. 알고리즘 조건

  • 입력 : 외부에서 제공되는 자료가 존재해야함
  • 출력 : 2가지 이상의 다른 결과를 출력
  • 명확성 : 명확한 명령어로 구성해야함 (코드로 작성할 수 있을 만큼 명확)
  • 유한성 : 유한한 시간안에 종료되어야함. 그래야 출력이 나오니까.
  • 효과성 : 사람이 종이와 연필로 수행할 수 있을 정도로 단순

2. 알고리즘 필요성

  • 서비스의 규모
  • 컴퓨터 연산 & 비용 (클라우드)

3. Python

  • glue 언어 : 다른 언어로 구현된 걸 풀로 붙이듯 사용할 수 있음
  • OOP, FP(람다 문법을 사용) 모두 지원
  • 코딩테스트에서 Comprehension, lambda 등 간결한 코드 작성 가능

[코딩테스트]

1. 목적

  • 코딩 습관

    • 알고리즘을 모르면 절대로 못 푸는 문제

Problem Solving - 카드 정렬하기(G4)

1. 문제 [Link]

정렬된 두 묶음의 숫자 카드가 있다고 하자. 각 묶음의 카드의 수를 A, B라 하면 보통 두 묶음을 합쳐서 하나로 만드는 데에는 A+B 번의 비교를 해야 한다. 이를테면, 20장의 숫자 카드 묶음과 30장의 숫자 카드 묶음을 합치려면 50번의 비교가 필요하다.

매우 많은 숫자 카드 묶음이 책상 위에 놓여 있다. 이들을 두 묶음씩 골라 서로 합쳐나간다면, 고르는 순서에 따라서 비교 횟수가 매우 달라진다. 예를 들어 10장, 20장, 40장의 묶음이 있다면 10장과 20장을 합친 뒤, 합친 30장 묶음과 40장을 합친다면 (10 + 20) + (30 + 40) = 100번의 비교가 필요하다. 그러나 10장과 40장을 합친 뒤, 합친 50장 묶음과 20장을 합친다면 (10 + 40) + (50 + 20) = 120 번의 비교가 필요하므로 덜 효율적인 방법이다.

N개의 숫자 카드 묶음의 각각의 크기가 주어질 때, 최소한 몇 번의 비교가 필요한지를 구하는 프로그램을 작성하시오.

2. 입력

첫째 줄에 N이 주어진다. (1 ≤ N ≤ 100,000) 이어서 N개의 줄에 걸쳐 숫자 카드 묶음의 각각의 크기가 주어진다. 숫자 카드 묶음의 크기는 1,000보다 작거나 같은 양의 정수이다.
