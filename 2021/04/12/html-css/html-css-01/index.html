HTML/CSS/JavaScript와 웹 표준, 접근성

Posted on 2021-04-12 In HTML/CSS

HTML, CSS, JavaScript

웹에서 동작하는 3가지 언어들

HTML

페이지에 제목, 문단, 표 이미지 등을 정의하고 구조와 의미를 부여하는 정적인 언어

CSS

Cascading Style Sheets 콘텐츠 구조를 꾸며주는 언어 웹의 시작적 표현을 담당.

JavaScript

컨텐츠를 동적으로 꾸며주는 역할을 하는 언어

  • HTML, CSS 역할도 할 수는 있지만, 동적인 업무에만 치중하는게 중요

웹 표준

웹 표준(Web Standard)이란 ‘웹에서 사용되는 표준 기술이나 규칙’

  • W3C의 권고안에서 나온 기술들이 여기에 해당

  • 브라우저를 만드는 업체(구글, MS, 애플 같은)에서 표준 기술을 해석하는 차이, 새로운 기술 삽입(표준화 제정 단계에 따른) 등으로 조금은 다르게 구동되는 브라우저가 생기게 됨

  • 대부분의 경우 표준화 제정 단계의 ‘권고안(REC)’ 에 해당하는 기술을 표준이라고

크로스 브라우징

조금은 다르게 구동되는 여러 브라우저에서 동일한 사용자 경험(같은 화면, 같은 동작 등)을 줄 수 있도록 제작하는 기술, 방법

  • IE에서도 동작하게 하는 것을 크로스 브라우징이라고 부르기도 합니다.(대부분의 경우 IE에서 문제가 없으면 다른 브라우저는 OK!

웹 접근성

일반 사용자부터 고령자, 장애인 같은 신체적, 환경적 조건에 제한이 있는 사용자를 포함해 모든 사용자들이 동등하게 접근할 수 있는 웹 콘텐츠를 제작하는 방법

  • 자막을 넣는 것
  • 키보드로만 사용가능한 웹
  • 대체 텍스트 등
  • 팀 버너스 리 경: 웹의 창시자

정보 통신 보조기기

장애인의 경우 다음과 같은 정보 통신 보조기기를 통해 웹 콘텐츠에 접근

  • 개인 구매 혹은 정부 기관이나 장애인 복지시설에서 임대하여 사용

웹 접근성 품질인증 마크

장애인 및 고령자가 웹 사이트 이용에 불편이 없도록 웹 접근성 표준을 준수한 우수 사이트에 대해 품질을 인증

  • 과학기술정보통신부가 지정한 웹 접근성 품질인증 기관
  • 페이지 수에 따라 기본 100만원 이상의 심사 비용을 지불
