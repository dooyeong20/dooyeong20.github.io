마크업

논리성이 중요함

  • 마크업은 논리적으로 와닿을 수 있도록 하는 것이 중요
  • 디자인을 보고 그대로 따라서 마크업 하는 것이 아니고, 실제 사용을 염두해두고 논리적인 순서에 따라 마크업하는 습관
  • 디자인 외적으로 최대한 semantic하게 마크업 해야 함

WAI-ARIA

웹 표준, 접근성의 중요성

  • fontawesome: aria-hidden=”true”
  • display: none 보다는 접근성을 조금 더 고려한 aria-hidden을 이해해야 한다.

nav tag

메뉴라고 다 네비게이션을 사용하지 말고, 실제로 중요한 메뉴에만 nav 태그를 사용한다.

a tag와 aria

button 태그를 사용하지 않고, a 태그를 버튼으로 쓰고자 한다면 다음 사항을 고려해야 함

  • a 태그에 role=”button” 을 주어서 스크린 리더가 버튼으로 인식하도록 함
  • aria-pressed=”true” : 눌렸을 때
  • aria-haspopup=”true”: 뭔가를 가졌는지

컨텐츠 숨기기

릴반적으로 position: absolute; top:-10000px 혹은 display: none 과 같은 방법으로 컨텐츠를 숨기지만 이는 접근성 측면에서 좋은 방법이 아니다.
다음과 같은 방법을 통해서 컨텐츠를 숨긴다면, 스크린 리더가 해당 컨텐츠를 읽을 수 있으며, 확면단에서 감출 수 있으므로 좋은 방법 중 하나라고 할 수 있다.

박스 border 표시하기

컨텐츠의 하단혹은 상단 등 선을 그어야 할 때 보통 border 프로퍼티를 활용한다. 하지만 box-shadow 속성을 통해서도 이를 충분히 구현할 수 있다.

float 해제하는 방법들

위 방식은, 사람들에게 많이 알려지지 않은 flow clear 방식이다. 보통은 float 자식들의 마지막 요소에 `clear: left | right | both 등으로 float을 해제한다.
