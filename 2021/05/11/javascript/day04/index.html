JavaScript 04 - 데이터 타입

작성일 2021-05-11 Edited on 2021-05-14 In JavaScript , Study

데이터 타입

데이터 타입이란 값의 “종류”를 말하는 것이다. 123 은 숫자 타입이고(참로고 자바스크립트에는 정수형이나 실수형이 구분되지 않고 모두 실수로 처리된다), hello 는 문자 타입이다.

원시 타입(primitive type)

자바스크립트에는 총 6개의 원시 타입이 존재한다

  • 숫자 타입 (number): 정수와 실수를 구분하지 않고 숫자 타입으로 존재

  • 문자열 타입 (string): 문자열

  • 불리언 타입 (boolean): 논리적 참과 거짓 (true/false)

  • undefined 타입: undefined var 키워드로 선언된 변수에 암묵적으로 할당되는 값

  • null 타입: 값이 없다는 것을 의도적으로 명시하는 값

  • 심볼(symbol): ES6 추가된 타입

그리고 객체 타입(object/reference type)

헷갈리지 않도록 하기 위해 예시를 하나 들겠다.

위에서 one은 숫자 타입, two는 문자열 타입이다.

각 타입별 특징

Number 타입

숫자 타입엔 특별한 세 가지 값들이 존재한다.

  • Infinity
  • -Infinity
  • NaN

자바스크립트는 대소문자를 구분하기 때문에 nan 과 같이 대소문자를 무시하고 사용할 수 없다는 점을 주의해야 한다.

String 타입

일반적으로 " ... " ' ... ' 와 같이 큰 따옴표와 작은 따옴표로 감싸는 방법 외에도 백틱(`) 으로도 감쌀 수 있는데, 문자열 표현식을 삽입하는 경우에 매우 편리하고 가독성이 좋다.

위와 같이 사용할 수 있으며 가독성이 좋은 것을 확인할 수 있다.

undefined 타입

자바스크립트를 배우며 처음에 undefined 타입과 null 을 구분하지 못해 헤맸었다. 하지만 이제는 확실히 정리해 헷갈리지 않도록 할 것이다.

undefined 는 var 키워드로 선언한 변수가 암묵적으로 초기화되는 값이다. 즉, 메모리 공간을 할당받고 초기화가 되기 전까지 빈 상태로 있으면 쓰레기 값이 존재하게 되는데, 이러한 값을 남겨두지 않고 undefined 로 초기화 하는 것이다.

이렇듯 undefined 는 개발자가 의도적으로 할당해주기 위한 값이 아니라 자바스크립트 엔진이 변수를 초기화할 때 사용하는 값이기 때문에 개발자가 직접 undefined 를 사용하는 것은 좋지 않다. 그러한 경우에는 null 을 대신 사용하는 것이 좋을 것이다.

DOM 에서 없는 요소를 선택했을 때는 undefined 가 아니라 null 값을 리턴하게 되는데 이 부분은 위 내용과 일관되지 않는 성격을 띄고 있기 때문에 주의해야 한다.

데이터 타입의 필요성

자바스크립트는 Java나 C 처럼 명시적으로 데이터 타입을 지정해주지 않는 동적인 언어이다. 그럼 이러한 동적인 언어인 자바스크립트에서는 어차피 데이터 타입을 자동으로 해석해줄텐데 왜 데이터 타입이라는 개념이 필요할까?

그것은 컴퓨터가 메모리에 자료를 어떻게 저장하고 사용하는지에 대한 내용과 밀접하다. 아무리 자체적으로 변수의 타입을 측정해준다고 한들, 실제로 메모리에 저장해주는 단계에서는 해당 자료(값)을 메모리 어떠한 공간에 얼만큼 사용해야 하는지 또 사용할 때에는 어떻게 해석해야 하는지 등을 알아야 하기 때문이다.

위 코드를 보면 age 라는 변수에 25 라는 숫자 타입의 값을 대입하려고 하고 있다. 이 때 숫자 타입은 8 바이트만큼을 차지하기 때문에 메모리에 공간을 할당하기 위해 8 바이트만큼의 공간을 확보해야 한다는 사실을 알아야한다.그리고 age 라는 변수를 사용할 때 변수가 참조하고 있는 메모리의 주소는 알지만, 한 번에 얼만큼의 바이트를 읽어와야 정보의 손실이 없을지 정확하게 알아야 하고 지금의 경우에는 8 바이트가 된다. 그리고 이렇게 읽는 것 까지 완료했다면 이제 읽어들인 정보를 어떻게 해석할지 알기 위해 데이터 타입이 필요하게 되는 것이다.

정리하자면

  • 값을 저장할 때 확보해야 하는 메모리 공간의 크기를 결정하기 위해
  • 값을 참조할 때 한 번에 읽어 들여야 할 메모리 공간의 크기를 결정하기 위해
  • 메모리에서 읽어 들인 2진수를 어떻게 해석할지 결정하기 위해

데이터 타입이 필요한 것이다.
