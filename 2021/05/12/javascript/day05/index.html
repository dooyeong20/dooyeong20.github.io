작성일 2021-05-12 Edited on 2021-05-14 In JavaScript , Study

제어문

제어문은 어떠한 프로그래밍 언어든 아주 중요한 역할을 한다. 주어진 조건에 따라서 특정한 코드 블록을 실행하거나 반복할 때 사용하는데, 이렇게 제어를 통해 코드의 실행 흐름을 제어할 수 있다. 하지만 흐름을 제어한다고해서 다 좋은 것은 아니고 코드의 흐름이 복잡해져 가독성 측면에서 좋지 않을 수도 있기 때문에 제어문을 남발하면 안된다.

조건문

조건문은 위 예시에서와 같이 if, else, switch 라는 키워드를 통해 코드블록의 실행을 결정하는 것이다.

이 때 조건식의 자리에 표현식이 들어올 때는 그 값이 만약 boolean 값이 아니더라도 암묵적으로 형변환이 수행되어 true, false 값으로 평가되게 된다.

반복문

반복문도 프로그래밍의 흐름 제어에서 아주 중요한 역할을 하고 시간 복잡도에 영향을 주는 메인 요소이다. 조건에 따라 일정한 코드 블록을 반복적으로 수행하는 것이고, 이 때 조건을 잘 체크하여 무한루프가 형성되지 않도록 하는 주의가 필요하다.

자바스크립트에서는 기본적으로 for, while, do while 문을 제공한다.

자바스크립트는 forEach, map, filter 등 다양한 메서드들이 존재하며 반복문처럼 Array의 각 요소를 iterate 할 때 사용된다.

break

이 때 while 문에서 조건식에 true 가 되는 값을 주어 무한루프를 만든 후, 반복문 내부에서 조건식을 만족했을 때만 루프가 종료되도록 if 조건문으로 break 키워드를 사용해 반복문을 빠져나올 수 있다.

그리고 이번에 새로 알게된 점은 반복문을 포함한 특정 코드블록에 label 을 달아놓아 해당 label 반복문을 선택적으로 탈출할 수 있다는 점이다.

그리고 꼭 반복문이 아니더라도 특정한 코드블록을 탈출할 수 있다.

continue

break 키워드와 반대로, continue 는 실행 중인 코드 블록의 위치에서 실행을 중단하고 탈출하는 것이 아니라, 바로 다음 단계의 반복으로 넘어가도록 흐름을 제어하는 키워드이다.

위 코드처럼 i 가 만약 홀수라면 continue 키워드를 통해 반복문을 종료하는 것이 아니라 바로 다음 단계의 반복 단계인 i + 1 번째로 넘어가서 다시 반복문을 수행하도록 하게 만드는 것이 continue 이다.

따라서 만약 반복문 내부에서 if 문이 필요할 때, 해당 조건문이 너무 길어진다면 continue 를 사용해 if 조건문에 해당하지 않을 때를 제외해주는 것이 코드상 더 깔끔할 것이다.
