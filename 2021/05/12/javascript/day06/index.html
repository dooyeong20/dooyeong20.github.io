타입 변환

자바스크립트의 값은 타입이 존재한다. 그리고 이러한 타입들간에 타입을 변환하는 것이 가능한데, 명시적 타입 변환(타입 캐스팅)과 암묵적 타입 변환이 있다.

위 코드는 명시적으로 개발자가 직접 의도를 가지고 타입을 변환해준 것이고 자바스크립트 엔진이 런타임에 자동으로 타입을 변환해주는 암묵적 타입 변환이 있다.

왜 숫자형과 빈 문자열을 더하면 문자열이 될까?

자바스크립트 엔진은 표현식 x + ''를 평가하기 위해서 새로운 문자열 값인 10 을 생성하고 그 다음 10 + '' 를 평가하게 된다. 그리고 그 결과로 '10' 이라는 값이 도출이 되어 결과적으로 x + '' 가 '10' dl 이 되는 것이다.

(Symbol()) + ‘’ 는 정상적으로 동작하지 않는데, 그 이유는 Symbol 값을 string 형으로 변환할 수 없기 때문이다.

그리고 조금 헷갈릴 수 있는 객체 타입의 경우에는 아래와 같이 변환할 수 있다.

그리고 불리언으로의 암묵적 형 변환은 조건문(반복문)에서 헷갈릴 수 있기 때문에 주의해서 사용해야하며, 다음과 같이 Truthy한 값과 Falsy한 값으로 구분할 수 있다.

  • Falsy

    • false
    • undefined
    • null
    • 0, -0
    • NaN
    • ‘’ (빈 문자열)

  • Truthy

    • 위 값들을 제외한 모두

명시적인 타입 변환이든 암묵적인 타입 변환이든 사실 중요한 것은 개발자가 코드가 어떻게 동작할지 정확하게 예측할 수 있어야 한다는 점이다. 만약 암묵적으로 타입이 변할 수 있다는 사실을 모르고 개발을 하다보면 암묵적 타입변환 때문에 디버깅에 많은 시간을 허비할 수 있기도 하고, 예상치 못한 에러가 발생할 수 있기 때문이다.

단축 평가

앞서 알아본 논리 연산자 부분에서 or 연산(||)과 and 연산(&&)에 대해서 알아보았는데, 이 연산의 결과는 꼭 boolean 값이 아닌 점에 주의해야 한다.

위 코드의 결과는 lee 이다. 왜 그럴까?

간단하게 결론부터 말하면, 가장 나중에 평가되는 값을 리턴한다고 생각하면 된다.

예를 들어서 && 연산의 경우 두 값 모두 true 여야 한다. 따라서 만약 좌항이 false 라면 우측 항은 볼 필요도 없이 결과가 false 일 것이다. 그리고 만약 좌측항이 true 일 경우, 우측항이 true 이면 true, false 이면 false 값이 반환될 것이다. 즉, 마지막 항의 값이 결과가 되는 것이다.

|| 연산의 경우, 좌측 항이 true 일 경우 항상 true 이기 때문에 좌측 항(true)을 그대로 반환하고, false 일 경우에는 우측 항이 true 일때 true, false 일 때 false 를 반환하기 때문에 우측 항의 값을 반환해주면 되는 것이다.
