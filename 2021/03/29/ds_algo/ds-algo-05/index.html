    이전 강의 복습

    • ‘is’ 와 ‘==’

      • == : 값을 비교해서 같으면 True
      • is : 동일한 객체인지를 확인해서 같으면 True

    • 참고로, 파이썬에서는 id()가 유일학 객체를 구분하는 수단(주소값은 알 수 없음)

    • 파이썬은 모든 값이 다 객체임. 모두.

      • 원시타입이 없음

    • 재귀보다 스택(반복문)이 성능이 훨씬 좋다. 하지만 읽기가 좀 힘들다.

    이진 탐색 트리(Binary Search Trees)

    • 노드 값에 ‘순서’, ‘크고 작음’이 있어서 비교를 할 수가 있다.(알파벳, 숫자 등도 포함)
    • 왼쪽 서브트리가 더 작고 오른쪽 서브트리 요소(노드)들이 더 크다.
    • 자료를 넣는 순서에 영향을 받는다.
    • O(logN) ~ O(n) 사이 어딘가에 존재하게 된다.
    • 자료의 중복이 허용되지 않음
    • (굳이 따지자면) 탐색 최적화

    자료의 탐색

    • Skewed 경우 : O(n)
    • Balance 경우 : O(logN)

    자료의 삽입

    • 일치하는 노드가 발견되면 종료
    • 일치하는 노드가 없으면 마지막 노드의 좌(작을 때), 우(클 때)에 삽입.

    자료의 삭제

    세가지 경우가 존재함
    삭제를 하기 위해 search를 할 때는 부모노드를 추적해야 한다.

    1. 삭제할 노드의 자식이 없을 때
    2. 삭제할 노드가 좌/우 중 하나의 자식이 있는 경우
    3. 삭제한 노드가 좌, 우 자식 모두 가지고 있는 경우 (매우 복잡)

    그래프

    객체간의 짝을 이루는 구조를 나타냄. 가장 유연하다 (ADT 임)

    • vertex와 이를 잇는 edge로 이루어져 있다.
    • 방향이 있거나(유유) 없을(무향) 수 있다.
    • edge에는 가중치라는 것이 있다.(클수록 연결의 강도가 클 수도있고, 작아질 수도 있다.)

    그래프와 트리 관계

    트리는 DAG(Directed Acyclic Graph)의 일종이며 회로(사이클)이 없고, 방향성이 존재한다.

    그래프의 종류

    • 무향 그래프(양쪽으로 다 연결되어 있는 것으로 생각할 수 있다.)
    • 유향 그래프
      • 같은 노드를 잇더라도 방향에 따라서 가중치가 다를 수 있다.
    • 다중 그래프
      • 하나의 vertex에서 다른 vertex로 가는 edge가 여러 개 있을 수 있다
      • 자기 자신으로 연결되는 edge가 있을 수 있다.

    그래프 용어

    • path: edge들의 시퀀스
    • 회로(Cycle, Circuit): 트리와는 다르게 사이클이 존재할 수 있다. 자기 자신으로 다시 돌아올 수 있음.
    • 부분 그래프(Subgraph): 그래프에 속하는 그래프
      • 모든 노드끼리 연결이 안되어 있어도 그래프임 즉, 1-7-3 6 4 이렇게 (1,7,3),(6), (4) 이렇게 구성해도 connected graph가 아닌 subgraph이다.
    • 연결 그래프(connected graph): 임의의 노드와 임의의 노드를 연결하는 edge가 있다. 즉, 어떻게든 path를 구성해서 노드간에 이동할 수 있다.

    그래프 구현 방법

    • 인접 리스트: Sparse 할 경우 유리함
      • 각 vertex마다 연결된 vertex들을 쭉 리스트로 나열함
      • 인접 리스트에서 순서는 중요하지 않음
        • 트리는 left, right(순서)가 중요하지만 그래프는 중요하지 않음
    • 인접 행렬 - Dense할 경우 유리함
      • 노드들간에 연결이 다 되어 있을 수록 유리
      • 무향 그래프인 경우, 인접 행렬에서 row=colum 축에 대칭임
      • 가중치가 있을 경우 (v1, v2)에 1대신에 가중치를 적음
      • 행렬이 값으로 차있을 경우(dense) 효율적
